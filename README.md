# Async Task Service

Небольшой backend-сервис на Go для асинхронной обработки задач через очередь.

## Что умеет

Сервис позволяет:

- создавать задачи через HTTP API
- получать задачу по id
- смотреть список всех задач
- обрабатывать задачи в фоне с помощью worker'а
- менять статус задачи в процессе обработки

Поддерживаемые статусы задачи:

- pending
- processing
- done
- failed

## Стек

Асинхронный backend-сервис на Go для создания и обработки задач с использованием PostgreSQL и фонового worker'а.

## Архитектура

Проект разделён на несколько слоёв:

- handler — HTTP-обработчики (internal/http/handler)
- router — регистрация маршрутов (internal/http/router)
- service — бизнес-логика
- repository — работа с базой данных
- worker — фоновая обработка задач
- model — структуры данных
- config — конфигурация приложения
- app — инициализация зависимостей приложения

## Как работает обработка задач

Клиент создаёт задачу через `POST /tasks`  
Задача сохраняется в БД со статусом `pending`  
Фоновый worker периодически ищет `pending` задачи  
Найденная задача переводится в `processing`

После обработки задача получает статус:
- `done`, если выполнена успешно
- `failed`, если произошла ошибка

Для безопасного захвата задач используется PostgreSQL-механизм:

```text
FOR UPDATE SKIP LOCKED
```
